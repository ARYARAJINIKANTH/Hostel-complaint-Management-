# Hostel Complaint Management System

## Project Overview
The Hostel Complaint Management System is a web-based application that helps hostel students register complaints and track their complaint status. The system enables administrators to manage and resolve complaints efficiently.

## Problem Statement
In many hostels, complaints such as electrical issues, plumbing problems, internet connectivity issues, and cleaning-related problems are handled manually. This can result in delays and difficulty in tracking complaint status. The proposed system provides a centralized platform for complaint registration and management.

## Technologies Used

### Frontend
- HTML
- CSS
- JavaScript

### Backend
- Python (Flask)

### Database
- PostgreSQL

## Modules

### Student Module
- Registration
- Login
- Submit Complaint
- View Complaint Status
- Complaint History

### Admin Module
- Login
- View Complaints
- Update Complaint Status
- Manage Complaints

## Significance
This system helps improve communication between students and hostel administration, reduces manual work, and ensures efficient complaint resolution.

## Expected Outcome
Students can register and track complaints online, while administrators can manage complaints effectively through a centralized system.
 ## Project Objectives

1. To develop a web-based Hostel Complaint Management System.
2. To provide an online platform for students to register complaints.
3. To enable students to track the status of their complaints.
4. To help hostel administrators manage complaints efficiently.
5. To reduce manual paperwork and improve transparency.
6. To maintain a centralized database of complaints and their resolutions.
7. To improve communication between students and hostel administration.
8. To ensure timely resolution of hostel-related issues.
